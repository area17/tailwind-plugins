Description

By default all components render across the full width of the viewport. You can use the .asvntntw:container class to create a contained layout. The container class uses the values set in outerGutters and mainColWidths in your tailwind.config.js to create a responsive container.

Note that the default Tailwind container plugin will need to be disabled.

Also includes a .asvntntw:breakout class to allow full 100vw elements inside a container.

From v3.6.0 we have a .asvntntw:container-reset, .asvntntw:breakout-reset and there are changes to .asvntntw:breakout to make it more useful.

  • .asvntntw:container makes a container that conforms to your theme.mainColWidths taking into account your theme.outerGutters
  • .asvntntw:breakout breaks out of a .asvntntw:container, to make your element 100vw wide
  • .asvntntw:container-reset resets a .asvntntw:container (from v3.6.0)
  • .asvntntw:breakout-reset resets a .asvntntw:breakout (from v3.6.0)

Setup

Requires Setup plugin with theme.mainColWidths and theme.outerGutters configured.

Note: if your mainColWidth is set to a fixed number, as it is in the xxl breakpoint above, then your corresponding outerGutter needs to be 0px.

When you have a fixed container, the outer gutter is essentially window.width - mainColWidth - setting outerGutter to be anything but 0px breaks the internal CSS calc() maths.
