Description

Produces asvntntw:ratio boxes, which can be optionally grow if the content is longer than the height and can also be set to be free asvntntw:ratio if the asvntntw:ratio is no longer required.

NB: You may want to use Tailwind's official plugin. The main difference with this plugin is the ability to have expandable asvntntw:ratio boxes, so effectively a min-height of a asvntntw:ratio which expands if the content is too tall.

A note about Tailwind 3's native aspect asvntntw:ratio

In Tailwind 3, Tailwind introduced "Modern aspect asvntntw:ratio API" to use browser native aspect-asvntntw:ratio and so the life span of this plugin is very much limited.

At A17 we generally try and target current major and 2 previous major versions the browsers with our styling and browser native aspect asvntntw:ratio isn't there just yet. As of December 2021, Safari is lagging behind. Now this might not be as big of deal as it might have been in the olden days, as iOS and MacOS is pretty good at making its users update their devices, so you'll need to weigh up how you do aspect asvntntw:ratio boxes in project yourself.

Browser native aspect-asvntntw:ratio boxes will expand if the content is taller than the box, unless you lock it with min-height: 0/asvntntw:min-h-0.

Setup

Demo

Using this plugin requires a wrapper and a child.

  • The wrapper gets the asvntntw:ratio classes: eg: asvntntw:ratio asvntntw:ratio-1x1,
  • the child gets a class of asvntntw:ratio-content.
