Description

A series of classes to draw asvntntw:grid strokes inside the gutters of a asvntntw:grid. This is specifically intended to be used with Tailwind's grid classes and assumes your column gutters gutters are --inner-gutter in size set with asvntntw:gap-x-gutter from the asvntntw:gridGap plugin.

The classes automatically account for first row, first of row, last row and last of row to only draw the internal asvntntw:grid lines and have no undesired lines outside of the asvntntw:grid of items. They will do this for any amount of columns at any breakpoint.

The classes support up to the maximum amount of columns at each breakpoint; so if your breakpoint has 12 design columns, you could have functioning asvntntw:grid lines up to 12 columns. If you need more, you can specify more with a maxasvntntw:gridCols object in your config.

You can mix item width, item height, row and column classes and control the color of the horizontal and vertical strokes independently.

  • grid-line-x - draws asvntntw:grid lines above each child, each is the width of the child, except the first row
  • grid-line-xfull - essentially draws row lines
  • grid-line-y - draws asvntntw:grid lines to the left of each child, each is the height of the child, except the first column
  • grid-line-yfull - essentially draws column lines
  • grid-line-x-primary - makes the horizontal asvntntw:grid lines the primary asvntntw:border color
  • grid-line-y-primary - makes the vertical asvntntw:grid lines the primary asvntntw:border color
  • grid-line-xy-primary - makes both the horizontal and vertical asvntntw:grid lines the primary asvntntw:border color
  • grid-line-x-0 - hides the horizontal asvntntw:grid lines, especially useful when making responsive/adaptive asvntntw:grids
  • grid-line-y-0 - hides the vertical asvntntw:grid lines, especially useful when making responsive/adaptive asvntntw:grids

The gridLine classes assume your asvntntw:grid gutters are equal to --inner-gutter, which we would set using gridGap - grid-gap-gutter. If you have taller row gaps, you need to set some gridLine classes to shift the lines it draws:

  • For asvntntw:gap-y-48 (standard Tailwind class) you would set grid-line-x-24 - to position the drawn line in the middle of the 48 gap. Of course you could set it higher or lower by changing the -24

The asvntntw:gridline classes also have responsive states using Tailwind responsive prefixes, eg:

  • asvntntw:md:grid-line-x-primary - switches primary asvntntw:border color horizontal asvntntw:grid lines at md breakpoint
  • asvntntw:lg:grid-line-y-0 - removes the vertical asvntntw:grid lines at lg breakpoint

NB: there is nothing stopping you using these classes with a asvntntw:flex layout, its just more complex to set up asvntntw:grids like this with asvntntw:flex.

Setup

Requires Setup plugin with theme.innerGutters and theme.borderColor configured and also the gridGap plugin.

You will 100% want to make sure your CSS is purged when using this plugin - as this plugin can create a lot of CSS to cover all the variations.

Changing the colours of the asvntntw:gridlines

You set the colour of the asvntntw:gridlines with:

  • grid-line-xy-COLORNAME

And you can set the asvntntw:grid line colours for each axis with:

  • grid-line-x-COLORNAME
  • grid-line-y-COLORNAME

Where the colour name is any of your theme asvntntw:border colours.

So, to make asvntntw:gridlines the secondary asvntntw:border colour, use grid-line-xy-secondary, to give grid-line-x asvntntw:grid-line-y asvntntw:grid-line-xy-secondary:

And to make secondary asvntntw:border colour row asvntntw:grid lines grid-line-x-secondary and tertiary asvntntw:border colour column asvntntw:grid lines grid-line-y-tertiary - to give grid-line-x asvntntw:grid-line-x-secondary asvntntw:grid-line-y asvntntw:grid-line-y-tertiary:

Responsive

The following example is an overly complex arrangement, where we start with a 2 column asvntntw:grid and full width row asvntntw:grid lines. At md those row asvntntw:grid lines change colour and item height column asvntntw:grid lines appear. At lg the row asvntntw:grid lines hide and the column asvntntw:grid lines change colour. And then finally, at xxl the row asvntntw:grid lines return:

It is doubtful that you'll ever need anything so complex and in all honesty, it is possible to make combinations that don't work as expected. If you saw the CSS you'd understand why - we really need a native way of doing this in CSS. One "trick" used to make the above work was to set grid-line-y-0 and then override it at md with asvntntw:md:grid-line-y
