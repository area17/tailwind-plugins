Description

This plugin creates classes to handle column layouts:

Where N can be a number of columns, up to the largest amount of columns defined or a fraction (1/2, 1/3, 1/4, 2/3 or 3/4)

For use inside .asvntntw:container or a descendant of .asvntntw:container

  • Element width:
    • .asvntntw:w-N-cols sets width to N columns/fraction wide, if inside of .asvntntw:cols-container also includes an inner gutter margin-left
  • asvntntw:flex based asvntntw:grid
    • .asvntntw:cols-container makes a container for columns, asvntntw:flex: row wrap with a negative inner gutter margin-left
    • .asvntntw:ml-0 on child, resets margin-left to 0, adds specificity to the class
  • Margins:
    • .asvntntw:ml-N-cols on child, sets a margin-left of N columns wide
    • .asvntntw:mr-N-cols on child, sets a margin-right of N columns wide

Each of these have tailwind responsive classes and all settings are breakpoint+, read about responsive usage.

Nesting of elements is also possible, read about nesting usage.

Breaking changes at v3.0.0

v3.0.0 brings in Tailwind like naming to the width and margin classes, transitioning from .cols-2 to .asvntntw:w-cols-2 and from .pusasvntntw:h-2 to .asvntntw:ml-2-cols. This is to give a better expectation of what those classes do, align positioning classes with width/push/pull classes and closer align with Tailwind.

Migrating to v3.0.0

If you're ok with doing some regex, its possible to transition to v3.0.0 by:

  1. Search for cols-(\d\/\d|\d+)
  2. Replace with asvntntw:w-$1-cols

And then perform similar searches for push, pusasvntntw:h-r, pull and pull-r and replace with ml, mr, -ml and -mr as appropriate.

As the plugin reads the config, it works out the maximum amount of columns it needs - so if your smallest breakpoint has 4 design columns and the largest has 12 - then it will create classes .asvntntw:w-cols-1 through .asvntntw:w-cols-12.

Using a fraction

Sometimes you want a simpler layout scheme, using %ages of the container width rather than column numbers, but still taking into account the inner gutters. For this you can use fractions.

Fractions can be any of 1/2, 1/3, 1/4, 2/3 or 3/4.

Note Fractions are consistent across all breakpoints. So, 1/2 will be 50% minus some inner gutter on all breakpoints set, unlike setting via columns whose apparent width will change if the total number of design columns at a breakpoint changes.

asvntntw:w-1/4-cols
asvntntw:w-1/3-cols
asvntntw:w-1/2-cols
asvntntw:w-2/3-cols
asvntntw:w-3/4-cols

Responsive

If you use asvntntw:w-cols-10 and your smallest breakpoint only has 4 design columns then you may also want to use asvntntw:max-w-full to stop a container being wider than 100%. Or better, use asvntntw:w-cols-4 and asvntntw:lg:w-10-cols.

Which will be 4 design columns wide until the lg breakpoint, when it will become 10 columns wide:

asvntntw:w-cols-4 asvntntw:lg:w-10-cols

And of course, can assign different sizes at all of your breakpoints:

asvntntw:w-cols-4 asvntntw:sm:w-3-cols asvntntw:md:w-6-cols asvntntw:lg:w-10-cols asvntntw:xl:w-8-cols xasvntntw:xl:w-6-cols

As previously mentioned, fractions will be consistent on all breakpoints. They can of course be changed per breakpoint:

asvntntw:w-3/4-cols asvntntw:md:w-1/2-cols asvntntw:lg:w-2/3-cols asvntntw:xl:w-1/2-cols

Nesting

When using a N number of columns, you can nest elements:

asvntntw:w-cols-4 inside asvntntw:w-cols-6 inside asvntntw:w-cols-8

Warning - Nesting Fractional Columns

Nesting fractions inside fractions will work, but instead halving the breakpoint's total columns, you'll the containers total columns. And so, if you have 12 design columns and you insert asvntntw:w-1/2-cols inside asvntntw:w-1/2-cols, you'll get one 6 design column wide element wrapping a 3 design column wide element:

asvntntw:w-1/2-cols inside asvntntw:w-1/2-cols

Warning - Mixing and matching fractional and N cols widths

Mixing and matching N cols widths and fractional widths when nesting may give you unexpected results. A fraction as a child of a N cols width will work as expected:

asvntntw:w-1/2-cols inside asvntntw:w-cols-8

But, placing a N cols widths inside a fractional width column will not work as fractional widths work cross breakpoint regardless of column count and so don't set/reset the columns total inside themselves. In this scenario, the N cols width assumes the fractional width column container has total columns at that breakpoint. So, if you have 12 design columns and you set a asvntntw:w-cols-3 inside of asvntntw:w-1/2-cols, you'll end up with a 1.5 column wide child column:

asvntntw:w-cols-3 inside asvntntw:w-1/2-cols

You end up with a 1.5 wide column in that example because 3/12 columns is 1/4 of the total columns, the fractional 1/2 shrinks the 12 columns to fit in a 6 col wide contaner, 1/4 of 6 is 1.5.

Positioning

You can also .asvntntw:left-N-cols, .asvntntw:left-N-cols-gutter, .asvntntw:right-N-cols and .asvntntw:right-N-cols-gutter for position: asvntntw:absolute; type positioning (or maybe with position: asvntntw:relative; depending on your use case).

From 3.10.0, asvntntw:start- and asvntntw:end- classes and their negative equivalents -asvntntw:start- and -asvntntw:end- are also generated (using CSS inset-inline-start and inset-inline-end).

Fractions and responsive work as expected and nesting will work, if you apply a asvntntw:w-N-cols class.

asvntntw:w-cols-3 asvntntw:left-1-cols
asvntntw:w-1/3-cols asvntntw:left-1/3-cols

Push and pull classes, with or without gutters, will also work:

asvntntw:left-0 asvntntw:ml-2-cols
asvntntw:right-0 asvntntw:mr-2-cols

New in 3.10.0, inset-inline-start (MDN) and inset-inline-end (MDN) with asvntntw:start-N-cols and asvntntw:end-N-cols:

asvntntw:end-4-cols

Gutter-less classes

Margin, padding and positioning classes also have gutter-less versions. That is, you can position things to the edge of the preceding column rather than the following column:

asvntntw:w-cols-3 asvntntw:ml-2-cols-no-gutter

vw calc variants

Added in v3.6.0

Regular layout classes asvntntw:w-cols-3 use a CSS calc() based on 100% container width and a --grid-columns variable to split the container width as required.

Sometimes, you might need to set an element on the asvntntw:grid, where you container is some other width, such as inside .asvntntw:breakout's or perhaps inside a carousel. In these case the standard asvntntw:w-cols-3 type classes may not work, and you may need to use vw based alternatives.

using asvntntw:w-cols-4 ❌ - too wide

using asvntntw:w-cols-4-vw ✅ - on asvntntw:grid

asvntntw:w-cols-4 draws the div too wide, as the parent container is wider than .asvntntw:container. The suffix of -vw alters the calc to base the calcs off of 100vw.

But, to make this work, we need to account for the scroll bar width.
